# 第二部分：功能概述

## 2.1 核心能力

**核心能力**：调用 GPT Image 2（`gpt-image-2-vip`）模型生成图片

**功能列表**：
- **自定义分辨率**：支持任意像素尺寸（WxH），如 `1920x1080`
- **比例格式**：支持 `1:1`、`3:4`、`16:9`、`9:16`、`21:9` 等，自动映射最优像素
- **比例快捷词**：输入 `小红书`→3:4、`海报`→2:3、`手机壁纸`→9:16 等
- **参考图生图**：传递 base64 图片，保持角色一致性
- **大图自动压缩**：>8MB 的参考图自动复制到临时目录压缩，上传后清理，原图不动
- **角色肖像专用生成**：常子涵定制提示词模板，预设参考图
- **Pipeline 自动化**：支持 PLOC治愈手账涂鸦 等模板化工作流

## 2.2 触发条件

当用户发送以下任一模式时自动激活：

| 模式 | 示例 |
|------|------|
| `帮我生成图片：xxx` | 帮我生成图片：一只橘猫在草地上晒太阳 |
| `生成图片：xxx` | 生成图片：赛博朋克风格的城市夜景 |
| `画一张图：xxx` | 画一张图：水墨风格的山水画 |
| `画图：xxx` | 画图：可爱的小狐狸 |
| `生成图片「xxx」` | 生成图片「宫崎骏风格的龙猫」 |
| `生成图片 [描述] [尺寸] [参考图]` | 生成图片 财阀千金 2:3 /path/to/ref.png |

**关键词**：`生成图片` / `画图` / `画一张` + 描述文字

# 第三部分：调用方式

## 3.1 Python API（核心）

```bash
# 基本用法
python gpt_image_api.py "图片描述" [尺寸] [参考图路径]

# 带比例快捷词
python gpt_image_api.py "小红书文案配图" "3:4"

# 带参考图（保持角色一致性）
python gpt_image_api.py "财阀千金风肖像" 1024x1536 ~/Desktop/ref.png

# 带输出名前缀
python gpt_image_api.py "旅行照片" "3:4" /path/to/ref.jpg -n "新疆之旅"
```

## 3.2 Shell脚本（便捷入口）

```bash
./gen_image.sh "图片描述" [尺寸] [参考图路径]
```

## 3.3 Pipeline自动化

```bash
# PLOC手账涂鸦
python pipeline/engine.py run PLOC_治愈手账涂鸦 --input /path/to/photo.jpg

# 列出所有可用模板
python pipeline/engine.py list

# 带输出路径
python pipeline/engine.py run PLOC_治愈手账涂鸦 --input /path/to/photo.jpg --output /path/out.png
```

# 第五部分：强制执行规定

> ⚠️ 以下规定为**铁律**，必须无条件执行。

## 规定一：不可改动用户提示词

用户给什么描述就原封不动传给 API，绝不自行改写、删减、翻译或优化。

## 规定一（补充）：不得擅自修改固定提示词模板

所有固定提示词模板均为经过验证的稳定版本，**严禁以下行为**：

- 擅自修改、删减、替换模板内容
- 将通用模板自行"优化"为自以为更准确的版本
- 在模板中添加未经验证的描述词

**例外**：只有用户明确要求修改某个模板时，才能在用户确认后调整。

### 固定模板清单

| 模板名 | 用途 | 文件位置 |
|--------|------|---------|
| `PORTRAIT_IDENTITY_ANCHOR` | 人像身份锚定 | `gpt_image_api.py` |
| `PORTRAIT_IDENTITY_BLOCK` | 肖像提示词 | `portrait_prompt.py` |
| `identity_anchor` | Pipeline身份锚定 | `pipeline/` |
| `negative` | 负面约束段落 | `pipeline/` |

## 规定二：生成常子涵肖像必须带参考图

**严禁无依据的单图生成。**

| 编号 | 文件 | 说明 | 大小 | 优先级 |
|------|------|------|------|--------|
| ref1.png | `ref1.png` | 偏分刘海妆后自拍 | 1.9MB | 备选 |
| ref2.jpg | `ref2.jpg` | 淡妆正脸自拍 | 117KB | **首选** |
| ref3_side.jpg | `ref3_side.jpg` | 侧脸参考（鼻子/下巴线条） | 67KB | 辅助·非必须 |

**优先级规则**：
- 单张生成时 → **只用 ref2.jpg**（文件最小，最稳定）
- 双参考图时 → **ref2.jpg + ref3_side.jpg**
- ref1.png 仅在 ref2 效果不佳时备用

## 规定三：禁止侧脸生成

**严禁生成 90° 纯侧脸。**

| 角度 | 一致性 | 说明 |
|------|--------|------|
| 90° 纯侧脸 | 严重下降 | ❌ 禁止 |
| 30-45° 微侧脸 | ~83% | ⚠️ 可接受 |
| 0-20° 正面/微侧 | 90%+ | ✅ 最佳 |

## 规定四：比例门禁

```
正方形(1024x1024) → 拦截
                   → 要求用户确认比例
                   → 除非gates_confirmed已设置（Pipeline调用时）
```

## 规定五：大图自动压缩

- **触发条件**：参考图 >8MB
- **处理方式**：复制到 `/tmp` 压缩，原图不动
- **压缩策略**：quality递减 → resize缩放

## 6.4 Pipeline工作流

### PLOC_治愈手账涂鸦

**用途**：旅行/风景照片加手账装饰风格

**Step 1 — VLM识别画面元素**（强制，不可跳过）
```bash
cd ~/.openclaw/skills && uv run python ollama_vlm.py \
  "列出图片中可标注的元素" /path/to/photo.jpg
```

**Step 2 — 文案撰写**（根据识别结果，3-5条繁体大字）
- ✅ 正确示范：`花園時光 ✦ 玫瑰與少年 ✦ 城市綠洲`
- ❌ 错误示范：`黑Polo ✦ 白褲 ✦ 銀手表`

**Step 3 — 拼入模板 + 生成**
```
ins風格治愈手賬裝飾，白色纖細手繪線條勾勒畫面裡所有物體輪廓，
搭配虛線、小圓圈可愛箭頭指向景物，字體要大、要粗、要清晰明顯，
起筆收筆痕跡，鋼筆書寫質感，不改變原圖畫面、不改動原圖色調，
只增加白色手繪標注裝飾，溫柔日系氛圍感。
附上溫柔治愈繁體手寫大字註解：{标注文案}
```

# 第十一部分：系统能力演进分析

> **来源**：工程三要素 v2.0 PRD交付规范 · 第十一模块（必填）
> **评级**：满分100分，本节缺失最高扣20分

## 11.1 过去（立项前系统缺了什么）

### 能力缺口盘点

| 缺口 | 具体表现 | 导致的问题 |
|------|---------|-----------|
| **无标准化生图接口** | 每次生成需要手动调用API，参数不统一 | 生成质量不稳定，Prompt碎片化 |
| **角色一致性缺失** | 无参考图体系，跨次生成面部走样 | 常子涵肖像无法保持一致 |
| **比例门禁缺失** | 任意尺寸均可提交，无质量把关 | 大量正方形生成，适配性差 |
| **无压缩机制** | 大参考图直接上传，易超时 | API超时失败率高 |
| **测试覆盖率零** | 无单元测试，回归风险高 | 改动即可能破坏已有功能 |
| **无Pipeline自动化** | 多步骤工作流需要手动串联 | PLOC等模板化场景无法批量处理 |
| **固定模板无管控** | 身份锚定等关键模板可被任意修改 | 一致性规则形同虚设 |

### 根因分析

1. **单次调用思维**：只解决"能生成"的问题，未考虑"如何稳定生成"
2. **模板资产意识缺失**：核心 Prompt 段落未做版本管理和保护
3. **工程规范未建立**：无测试规范，代码质量依赖手动验证
4. **平台协作接口缺失**：OpenClaw 与外部 API 之间的桥接不完整

## 11.3 未来（新框架/新需求来了怎么办）

### 新挑战映射

| 新需求 | 当前架构能否应对 | 扩展方案 |
|--------|----------------|---------|
| **新增模型（如 Imagen 4）** | 部分可应对（model参数可换） | 扩展 `gpt_image_api.py` 的 `model` 参数支持 |
| **新增比例快捷词** | 可应对（修改 `VERTICAL_PRESETS` 即可） | 配置驱动，零代码 |
| **新增Pipeline模板** | 可应对（写JSON注册到 `registry.json`） | 模板热加载，无需改代码 |
| **新增人像风格** | 可应对（修改 `PORTRAIT_STYLE_TERMS`） | 配置驱动，零代码 |
| **接视频生成模型** | ❌ 需要新Skill | 独立 skill-video-generation |
| **接 Stable Diffusion 本地模型** | ❌ 架构不兼容 | 需新增 `sd_local.py` 独立模块 |
| **多语言 Prompt 翻译** | ❌ 无翻译层 | 未来可加 translation middleware |

### 架构评级

| 维度 | 评级 | 说明 |
|------|------|------|
| **配置驱动** | A | VERTICAL_PRESETS/PORTRAIT_STYLE_TERMS/registry.json 均配置化 |

| **整体架构** | **A级** | 配置驱动 + 函数内聚 + 测试规范，可扩展性强 |

**架构结论**：当前架构为**A级可扩展架构**。新增模型/模板/快捷词均无需修改核心逻辑，通过配置文件热加载即可完成扩展。主要扩展边界在于"跨模型类型"（如新增视频生成）和"本地模型"，这两类需要独立Skill。

# 第十二部分：FAQ

## 12.1 开发常见问题

**Q1: 如何获取API Key？**
A: 访问 https://yh.grsai.ai 注册获取。

**Q2: 如何配置API Key？**
A:
```bash
# 方式一：.env文件
cp .env.example .env
nano .env  # 填入真实 Key

# 方式二：环境变量
export GRSAPI_KEY=your-key
```

**Q3: 为什么正方形被拦截？**
A: 比例门禁规定，除正方形外都需要确认具体用途，以适配最佳比例。

**Q4: 如何跳过比例门禁？**
A: Pipeline调用时自动传入`gates_confirmed=_GATES_CONFIRMED`。

**Q5: 参考图路径如何指定？**
A: 直接传本地文件路径，如 `/path/to/ref2.jpg`。

**Q6: 大图压缩会影响原图吗？**
A: 不会。压缩副本在/tmp目录，原图位置、内容、名称均不改变。

**Q7: 固定模板可以修改吗？**
A: 不可以。固定模板（身份锚定、negative约束等）均为验证过的稳定版本，严禁擅自修改。

**Q8: 常子涵肖像必须用哪张参考图？**
A: 首选ref2.jpg（117KB，最稳定），次选ref2+ref3组合。

**Q9: 测试覆盖率如何达标？**
A: 运行 `uv run pytest tests/ -v --cov=. --cov-report=term-missing`，目标≥80%。三测全部PASS后上传仓库。

**Q10: Pipeline模板如何注册？**
A: 在 `pipeline/templates/registry.json` 中添加模板元数据，创建对应JSON文件即可热加载。

## 12.2 用户常见问题

**Q1: 生成需要多长时间？**
A: 通常15-30秒，受网络和队列影响。

**Q2: 积分不足怎么办？**
A: 访问 https://yh.grsai.ai 充值。

**Q3: 支持哪些尺寸？**
A: 任意宽×高均可，如`1024x1365`、`1920x1080`等，或使用快捷词`小红书`、`海报`等。

**Q4: 如何保持角色一致性？**
A: 携带参考图即可自动注入身份锚定，保持面部特征一致。

**Q5: 提示"Permission denied"？**
A: 运行`chmod +x gen_image.sh`赋予执行权限。
